# AGENTS Instructions

Этот файл задаёт локальные правила для любых AI-агентов и coding assistants, работающих с данным репозиторием.

## Commit Policy
- Используй conventional commits.
- Пиши сообщения коммитов на русском языке.
- Предпочтительные префиксы: `feat:`, `fix:`, `docs:`, `refactor:`, `chore:`.

## Changelog Policy
- Любой значимый коммит должен сопровождаться обновлением `CHANGELOG.md`.
- Под значимым коммитом понимается изменение, которое влияет на поведение системы, конфигурацию, интеграции, запуск, эксплуатацию, безопасность или документацию проекта.
- Если изменение вносится в код, `systemd` unit-файлы, env-шаблоны, инструкции по запуску или публичную документацию, агент обязан проверить, нужно ли обновить `CHANGELOG.md`.
- Если changelog не обновлён, агент не должен считать задачу завершённой без явной причины.

## Repository Hygiene
- Не коммить секреты, реальные env-файлы, логи, приватные заметки и локальные рабочие артефакты.
- Если существует риск утечки секрета, агент должен явно указать на это в результате работы.

## Documentation Policy
- При изменении поведения проекта агент должен обновить релевантные разделы `README.md`.
- Если добавляется новая конфигурация, переменная окружения или сценарий запуска, это должно быть отражено в документации.
