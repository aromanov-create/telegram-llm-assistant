# My Assistant Bot

Telegram-бот, который превращает Telegram в личную точку входа для повседневных задач через LLM.

Проект предназначен для сценария, где пользователю удобнее написать, надиктовать или прислать фото/видео в Telegram, а дальше бот:
- принимает запрос;
- при необходимости расшифровывает голос через Whisper;
- передаёт задачу в `claude` CLI;
- возвращает итоговый ответ обратно в Telegram.

## Зачем это вообще нужно

Идея сервиса простая: Telegram уже у большинства всегда под рукой, а значит он может быть удобным интерфейсом к персональному AI-ассистенту.

Вместо отдельного UI, браузерной вкладки или ручного запуска CLI-команд вы можете использовать обычный чат с ботом как универсальный вход для:
- быстрых поручений;
- разбора входящих материалов;
- голосовых заметок и черновиков;
- задач, которые проще сформулировать на ходу с телефона.

## Практические сценарии использования

Бот можно использовать не только как чат с LLM, а как мобильную точку входа в личные и рабочие автоматизации. Если `claude` CLI на машине уже умеет вызывать локальные скрипты, ходить в API или работать с файлами, Telegram-бот становится удобным интерфейсом ко всему этому.

Типовые кейсы использования:

- разбор и подготовка писем: кратко пересказать входящее письмо, подготовить черновик ответа, переформулировать текст под нужный тон;
- отправка писем: по команде пользователя собрать письмо и отправить его через подключённый почтовый API или локальный скрипт;
- работа с внешними API: сходить в CRM, task tracker, Notion, Google Sheets, внутренний сервис или любой HTTP API и вернуть результат в Telegram;
- постановка и обновление задач: создать задачу, найти текущие задачи, поменять статус, добавить комментарий, собрать краткий daily summary;
- работа с документами: принять фото, текст или файл, извлечь главное, сделать выжимку, подготовить черновик ответа или список действий;
- обработка голосовых заметок: надиктовать мысль на ходу и получить структурированный текст, TODO-список или готовое поручение;
- анализ входящих материалов: прислать фото интерфейса, документа, таблицы или видео и получить краткое объяснение, что там происходит;
- интеграция с локальной автоматизацией: запускать свои shell-скрипты, ETL, парсеры, отчёты и сервисные команды прямо из Telegram;
- мониторинг и операционные действия: проверить статус сервиса, собрать диагностическую информацию, перезапустить процесс или запросить сводку по логам;
- персональный workflow-хаб: использовать бота как единый интерфейс ко всем своим повседневным действиям, которые неудобно каждый раз делать вручную с ноутбука.

Примеры более конкретных запросов:

- "проверь новые письма и кратко перескажи только важные";
- "подготовь ответ на это письмо в деловом тоне";
- "отправь Сергею письмо с итогами встречи";
- "создай задачу в трекере: подготовить КП к пятнице";
- "покажи мои открытые задачи по проекту X";
- "сходи в API и покажи последние лиды за сегодня";
- "посмотри это фото документа и выпиши ключевые пункты";
- "сделай из этой голосовой заметки нормальный план действий";
- "проверь, жив ли сервис, и если нет — перезапусти";
- "собери краткий отчёт за день по заявкам, письмам и задачам".

Именно в этом и ценность такого проекта: Telegram здесь выступает не просто как мессенджер, а как универсальный интерфейс к почте, API, файлам, локальным скриптам и рабочим процессам пользователя.

## Что умеет бот

Бот принимает:
- текстовые сообщения
- голосовые сообщения
- фото
- видео
- файлы любого типа (txt, pdf, docx, py, изображения как документ и т.д.)

Голос расшифровывается через Whisper, после чего запрос передаётся в `claude` CLI. Для фото и видео бот передаёт Claude путь к локальному файлу или извлечённому кадру. Файлы скачиваются во временную директорию, Claude читает их через свои инструменты (изображения — через vision, текстовые файлы — через Read).

Во время выполнения любого запроса отображается кнопка **«❌ Отменить»** — нажатие мгновенно прерывает запущенный процесс.

### Команды

- `/new` — очистить историю разговора и начать заново
- `/start` — информация о боте и идентификатор пользователя

## Как это устроено

В проекте есть два user-level systemd сервиса:

- `assistant-bot.service` — основной бот
- `ssh-proxy-tunnel.service` — пример unit-файла для SSH SOCKS5 туннеля

`assistant-bot.service` стартует из директории проекта, читает настройки из `assistant-bot.env` и работает с автоперезапуском. Прокси не зашит в unit-файл: при необходимости его можно задать через переменные окружения в `assistant-bot.env`.

В коде дополнительно есть watchdog: если Telegram API недоступен слишком долго, процесс завершается, а `systemd` поднимает его заново.

## Структура

- `bot.py` — основной код бота
- `assistant-bot.service` — unit-файл user service для бота
- `ssh-proxy-tunnel.service` — пример unit-файла для SSH туннеля
- `assistant-bot.env.example` — пример env-файла
- `assistant-bot.env` — локальный env-файл с секретами, не коммитится

## Конфигурация

Скопируйте шаблон env-файла:

```bash
cp assistant-bot.env.example assistant-bot.env
```

Заполните `assistant-bot.env`:

```env
BOT_TOKEN=<telegram bot token>
ALLOWED_USER_ID=<telegram user id>

# Optional proxy settings
# http_proxy=http://127.0.0.1:8118
# https_proxy=http://127.0.0.1:8118
# HTTP_PROXY=http://127.0.0.1:8118
# HTTPS_PROXY=http://127.0.0.1:8118
```

Права на файл должны быть ограничены:

```bash
chmod 600 assistant-bot.env
```

Если прокси не нужен, строки `http_proxy` / `https_proxy` можно не добавлять вовсе. Если нужен свой прокси, просто укажите его в `assistant-bot.env` без правок `assistant-bot.service`.

## Требования

На машине должны быть доступны:

- `python3`
- Python-библиотеки, которые использует `bot.py`
- `ffmpeg`
- `claude` CLI
- при необходимости HTTP(S)-прокси, который можно указать в `assistant-bot.env`
- настроенный SSH host alias, если вы используете `ssh-proxy-tunnel.service`

## Установка systemd service

Скопировать unit-файлы в user systemd:

```bash
install -m 644 assistant-bot.service ~/.config/systemd/user/assistant-bot.service
install -m 644 ssh-proxy-tunnel.service ~/.config/systemd/user/ssh-proxy-tunnel.service
systemctl --user daemon-reload
```

Включить автозапуск:

```bash
systemctl --user enable ssh-proxy-tunnel.service
systemctl --user enable assistant-bot.service
```

Запустить:

```bash
systemctl --user start ssh-proxy-tunnel.service
systemctl --user start assistant-bot.service
```

## Полезные команды

Статус сервисов:

```bash
systemctl --user status assistant-bot.service
systemctl --user status ssh-proxy-tunnel.service
```

Рестарт бота:

```bash
systemctl --user restart assistant-bot.service
```

Логи бота:

```bash
journalctl --user -u assistant-bot.service -f
```

Логи туннеля:

```bash
journalctl --user -u ssh-proxy-tunnel.service -f
```

Проверка синтаксиса Python:

```bash
python3 -m py_compile bot.py
```

## Ручной запуск без systemd

Для локальной проверки можно запустить напрямую:

```bash
set -a
source assistant-bot.env
set +a
python3 bot.py
```

Но основной режим эксплуатации для этого проекта — через `systemd`.
